1. Introduction

본 보고서는 이전 단계인Conceptualization Document에서 정의한 요구사항을 바탕으로 진행되는Analysis Phase의 결과물이다. 시스템의 구조를 논리적으로 분석하는 두 번째 단계로서,서비스의 기능을Actor와Use Case의 유기적 관계로 정의한Use Case Diagram및Description을 포함한다. 또한, 시스템 내 주요 객체 간의 데이터를 정의하는Domain analysis와실제 구현에 앞서 사용자 경험을 가시화한User Interface Prototype을 함께 제시하여 개발의일관성을 확보하고자 한다.

1.1. Executive Summary

대부분의 현대인은 기상을 위해 모바일 알람을 사용하지만, 기상 직후 인지 능력이 저하된상태에서 무의식적으로 알람을 종료하고 다시 잠드는 '수면 관성' 문제를 빈번하게 겪는다. 기존알람 서비스는 단순히 소음을 반복하는 방식에 그쳐, 사용자의 실질적인 각성을 유도하거나기상에 대한 동기를 부여하기에는 한계가 명확하다. 이러한 불편함을 해소하고 확실한 아침기상을 돕기 위해,RPG게임의 전투 시스템을 도입하여 특정 미션을 해결해야만 알람이해제되는 게이미피케이션 앱인 '알람RPG:던전 탈출(Alarm Dungeon)'을 기획하게 되었다.

1.2. Business Goals

Alarm Dungeon의 궁극적인 목표는 사용자가 기상과 동시에 두뇌 활동을 시작하게 함으로써재수면을 방지하고 생산적인 아침 습관을 형성하는 것이다. 사용자는 알람이 울림과 동시에가상의 몬스터를 조우하며, 산수 풀이나 기기 흔들기 등의 퀘스트를 성공적으로 수행하여몬스터를 토벌해야 한다. 이 과정에서 제공되는 레벨업은 기상을 단순한 고역이 아닌 성취감있는 활동으로 치환한다. 불완전한 기상 방식으로 발생하는 시간적 손실과 일상의 피로도를최소화하여 사용자 삶의 질을 향상하는 데 목적이 있다.

1) User (사용자)

등록된 사용자의 계정 정보를 관리하는 클래스이다. 각 사용자는 고유한ID와 아이디로 인증되며,하나의 캐릭터를 소유한다.

주요 속성:- userId:사용자 고유ID - username:로그인용 아이디 -password:암호화된비밀번호 -createdDate:가입 날짜 -lastLoginDate:마지막 로그인 날짜 -characterId:소유한캐릭터ID (FK)

2) Character (캐릭터)

사용자가 플레이하는 게임 내 캐릭터를 나타내는 클래스이다. 매일의 알람 미션 완료를 통해경험치를 획득하고 레벨업한다.

주요 속성:- characterId:캐릭터 고유ID - userId:소유자ID (FK) - name:캐릭터 이름 - -level:현재 레벨 -experience:현재 경험치 -maxExperience:다음 레벨까지 필요한 경험치 -totalAwakeTime:누적 일어난 시간 -averageAwakeTime:평균 깨는 시간

3) Alarm (알람)

사용자가 설정한 알람 정보를 관리하는 클래스이다. 각 알람은 특정 시간에 울리고, 연동된퀴즈/미션을 실행한다.

주요 속성:- alarmId:알람 고유ID - userId:소유자ID (FK) - time:알람 시간 (HH:MM형식) -daysOfWeek:반복 요일 (비트마스크) -isEnabled:활성화 여부 -questId:연동 퀘스트ID (FK) -createdDate:생성 날짜

4) Quest (퀘스트/던전)

사용자가 알람 시에 수행할 미션의 집합을 나타내는 클래스이다. 각 퀘스트는 몬스터와의 전투로표현된다.

주요 속성:- questId:퀘스트 고유ID - missionType:미션 유형 (QUIZ/SHAKE/PHOTO) -difficulty:난이도 (EASY/NORMAL/HARD) - requiredExperience:완료 시 획득 경험치 -createdBy:관리자ID (FK)

5) Mission (미션)

퀘스트에 포함된 개별 미션을 나타내는 클래스이다. 퀴즈, 센서 기반 작업 등 다양한 유형을가진다.

주요 속성:- missionId:미션 고유ID - questId:소속 퀘스트ID (FK) - type:미션 유형(QUIZ/SHAKE/PHOTO) - content:미션 내용 (문제, 요구사항 등) -answer:정답 (퀴즈의 경우) -targetCount:목표 값 (센서 미션의 경우)

6) Puzzle (퀴즈)

게임에 사용되는 퀴즈 문제를 관리하는 클래스이다. 관리자가 추가하고 수정할 수 있다.

주요 속성:- puzzleId:퍼즐 고유ID - question:문제 텍스트 -answer:정답 -options:객관식선택지 (List) - difficulty:난이도 (EASY/NORMAL/HARD) - createdBy:관리자ID (FK)

7) AlarmLog (알람 로그)

각 알람 실행 및 미션 완료 기록을 저장하는 클래스이다.

주요 속성:- logId:로그 고유ID - userId:사용자ID (FK) - alarmId:알람ID (FK) -executedTime:실행된 시간 -isSucceeded:성공 여부 -awakeTime:깨는데 걸린 시간 (초) -earnedExperience:획득한 경험치 -createdDate:기록 날짜
