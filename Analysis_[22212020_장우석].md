## 1. Introduction

본 보고서는 이전 단계인 Conceptualization Document에서 정의한 요구사항을 바탕으로 진행되는 Analysis Phase의 결과물이다. 시스템의 구조를 논리적으로 분석하는 두 번째 단계로서, 서비스의 기능을 Actor와 Use Case의 유기적 관계로 정의한 Use Case Diagram 및 Description을 포함한다. 또한, 시스템 내 주요 객체 간의 데이터를 정의하는 Domain analysis와 실제 구현에 앞서 사용자 경험을 가시화한 User Interface Prototype을 함께 제시하여 개발의 일관성을 확보하고자 한다. 

### 1.1 Executive Summary

대부분의 현대인은 기상을 위해 모바일 알람을 사용하지만, 기상 직후 인지 능력이 저하된 상태에서 무의식적으로 알람을 종료하고 다시 잠드는 '수면 관성' 문제를 빈번하게 겪는다. 기존 알람 서비스는 단순히 소음을 반복하는 방식에 그쳐, 사용자의 실질적인 각성을 유도하거나 기상에 대한 동기를 부여하기에는 한계가 명확하다. 이러한 불편함을 해소하고 확실한 아침 기상을 돕기 위해, RPG 게임의 전투 시스템을 도입하여 특정 미션을 해결해야만 알람이 해제되는 게이미피케이션 앱인 '알람 RPG: 던전 탈출(Alarm Dungeon)'을 기획하게 되었다. 

### 1.2 Business Goals

Alarm Dungeon의 궁극적인 목표는 사용자가 기상과 동시에 두뇌 활동을 시작하게 함으로써 재수면을 방지하고 생산적인 아침 습관을 형성하는 것이다. 사용자는 알람이 울림과 동시에 가상의 몬스터를 조우하며, 산수 풀이나 기기 흔들기 등의 퀘스트를 성공적으로 수행하여 몬스터를 토벌해야 한다. 이 과정에서 제공되는 레벨업은 기상을 단순한 고역이 아닌 성취감 있는 활동으로 치환한다. 불완전한 기상 방식으로 발생하는 시간적 손실과 일상의 피로도를 최소화하여 사용자 삶의 질을 향상하는 데 목적이 있다.

| 동시성 | 한 번에 하나의 미션만 실행 |

## 3. Domain analysis

#### 1) User (사용자)

등록된 사용자의 계정 정보를 관리하는 클래스이다. 각 사용자는 고유한 ID와 아이디로 인증되며, 하나의 캐릭터를 소유한다.

**주요 속성:**

- `userId`: 사용자 고유 ID
- `username`: 로그인용 아이디
- `password`: 암호화된 비밀번호
- `createdDate`: 가입 날짜
- `lastLoginDate`: 마지막 로그인 날짜
- `characterId`: 소유한 캐릭터 ID (FK)

#### 2) Character (캐릭터)

사용자가 플레이하는 게임 내 캐릭터를 나타내는 클래스이다. 매일의 알람 미션 완료를 통해 경험치를 획득하고 레벨업한다.

**주요 속성:**

- `characterId`: 캐릭터 고유 ID
- `userId`: 소유자 ID (FK)
- `name`: 캐릭터 이름
- `level`: 현재 레벨
- `experience`: 현재 경험치
- `maxExperience`: 다음 레벨까지 필요한 경험치
- `totalAwakeTime`: 누적 일어난 시간
- `averageAwakeTime`: 평균 깨는 시간

#### 3) Alarm (알람)

사용자가 설정한 알람 정보를 관리하는 클래스이다. 각 알람은 특정 시간에 울리고, 연동된 퀴즈/미션을 실행한다.

**주요 속성:**

- `alarmId`: 알람 고유 ID
- `userId`: 소유자 ID (FK)
- `time`: 알람 시간 (HH:MM 형식)
- `daysOfWeek`: 반복 요일 (비트마스크)
- `isEnabled`: 활성화 여부
- `questId`: 연동 퀘스트 ID (FK)
- `createdDate`: 생성 날짜

#### 4) Quest (퀘스트)

사용자가 알람 시에 수행할 미션의 집합을 나타내는 클래스이다. 각 퀘스트는 몬스터와의 전투로 표현된다.

**주요 속성:**

- `questId`: 퀘스트 고유 ID
- `name`: 던전 이름
- `monsterName`: 보스 몬스터 이름
- `missionType`: 미션 유형 (QUIZ/SHAKE/PHOTO)
- `difficulty`: 난이도 (EASY/NORMAL/HARD)
- `requiredExperience`: 완료 시 획득 경험치

#### 5) Mission (미션)

퀘스트에 포함된 개별 미션을 나타내는 클래스이다. 퀴즈, 센서 기반 작업 등 다양한 유형을 가진다.

**주요 속성:**

- `missionId`: 미션 고유 ID
- `questId`: 소속 퀘스트 ID (FK)
- `type`: 미션 유형 (QUIZ/SHAKE/PHOTO)
- `content`: 미션 내용 (문제, 요구사항 등)
- `answer`: 정답 (퀴즈의 경우)
- `targetCount`: 목표 값 (센서 미션의 경우)

#### 6) Puzzle (퀴즈)

게임에 사용되는 퀴즈 문제를 관리하는 클래스이다. 관리자가 추가하고 수정할 수 있다.

**주요 속성:**

- `puzzleId`: 퍼즐 고유 ID
- `question`: 문제 텍스트
- `answer`: 정답
- `options`: 객관식 선택지 (List<String>)
- `difficulty`: 난이도 (EASY/NORMAL/HARD)

#### 7) Monster (몬스터)

던전 퀘스트에 등장하는 몬스터 정보를 관리하는 클래스이다.

**주요 속성:**

- `monsterId`: 몬스터 고유 ID
- `name`: 몬스터 이름
- `questId`: 관련 퀘스트 ID (FK)
- `health`: 몬스터 체력
- `imagePath`: 몬스터 이미지 경로

#### 8) AlarmLog (알람 로그)

각 알람 실행 및 미션 완료 기록을 저장하는 클래스이다.

**주요 속성:**

- `logId`: 로그 고유 ID
- `userId`: 사용자 ID (FK)
- `alarmId`: 알람 ID (FK)
- `executedTime`: 실행된 시간
- `isSucceeded`: 성공 여부
- `awakeTime`: 깨는데 걸린 시간 (초)
- `earnedExperience`: 획득한 경험치

## 5. 용어 해설

| 용어 | 정의 |
|------|------|
| **게이미피케이션** | 게임이 아닌 분야(기상 알람)에 게임의 메커니즘, 규칙, 보상 등을 적용하여 사용자의 참여도와 동기를 높이는 기법 |
| **2차 수면** | 첫 번째 알람을 끈 후 무의식적으로 다시 잠드는 행위로, 본 프로젝트가 해결하고자 하는 핵심 문제 |
| **스누즈** | 알람을 몇 분 뒤에 다시 울리도록 미루는 기능 |
| **던전** | RPG 게임에서 플레이어가 도전하는 미션 스테이지. 본 프로젝트에서는 알람과 연동된 퀘스트를 의미 |
| **몬스터** | 각 던전에 등장하는 보스급 적으로, 사용자가 미션을 완료하면 처치 |
| **퀘스트** | 사용자가 알람 시간에 수행할 미션들의 모음으로, 특정 몬스터와 연동 |
| **미션** | 알람이 울렸을 때 사용자가 수행해야 하는 구체적인 작업 (퀴즈, 기기 흔들기, 사진 촬영 등) |
| **경험치 (EXP)** | 미션 완료 시 획득하는 포인트로, 일정량 모으면 캐릭터의 레벨이 상승 |
| **레벨** | 캐릭터의 성장 단계를 나타내는 지표 |
| **복셀** | 부피(Volume)와 픽셀(Pixel)의 합성어로, 3D 공간에서 입방체 형태로 표현되는 그래픽 단위 |
| **센서** | 모바일 기기의 카메라, 가속도계, 마이크 등 각종 감지 장치 |
| **동기화** | 모바일 클라이언트와 게임 서버 간의 캐릭터 데이터, 미션 기록 등을 일치시키는 과정 |
| **알람 로그** | 각 알람 실행 및 미션 완료 기록으로, 사용자의 일어난 시간 등을 추적 |
| **도감** | 처치한 몬스터들의 정보를 기록하는 시스템 |
| **일어난 시간** | 알람이 울린 후 미션을 완료할 때까지 소요된 시간으로, 사용자의 기상 품질을 측정하는 지표 |
